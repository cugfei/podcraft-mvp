# PodCraft 测试用例（基于 README.md 与 DEPLOYMENT.md）

## 1. 说明
- 文档依据：`README.md`、`DEPLOYMENT.md`
- 覆盖范围：MVP 核心功能（认证、播客创建、编辑合成）+ 本地运行验证 + Docker 部署验证 + 常见故障场景
- 不覆盖：第三方 TTS 供应商真实计费链路、生产域名 SSL 证书细节

## 2. 测试环境
- 前端：Next.js（默认 `http://localhost:3000`）
- 后端：FastAPI（默认 `http://localhost:8032`，或部署模式 `8000`）
- 数据库：SQLite（开发）
- 浏览器：Chrome 最新稳定版

## 6. 文档中“常见问题”回归用例

| 用例ID | 问题点 | 步骤 | 预期结果 | 优先级 |
|---|---|---|---|---|
| REG-CORS-001 | CORS 导致登录无响应 | 前端改用未配置端口启动（如 3001）后尝试登录 | 复现失败；将端口加入 `CORS_ORIGINS` 并重启后恢复 | P0 |
| REG-PORT-001 | 端口占用 | 占用 3000 后启动前端 | Next 自动切换端口（3001/3002...） | P1 |
| REG-DB-001 | 数据库初始化缺失 | 删除开发库后直接访问关键流程 | 报错可复现；执行 `alembic upgrade head` 后恢复 | P1 |
| REG-REDIS-001 | Redis 不可用 | 关闭 Redis 后触发依赖任务 | 出现可识别错误，恢复 Redis 后正常 | P2 |

## 7. 执行建议
- 冒烟顺序建议：`AUTH -> CREATE -> EDIT -> TTS -> PODCAST LIST`
- 每次发版至少执行：所有 `P0` + 关键 `P1`（创建、编辑、合成、部署）
- 缺陷记录建议字段：`用例ID / 版本 / 环境 / 实际结果 / 期望结果 / 日志截图`
